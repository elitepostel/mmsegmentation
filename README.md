
# Semantic Segmentation Project (mmsegmentation practicum scaffold)

Этот каркас предназначен для проекта по семантической сегментации в форке `mmsegmentation`.
Его нужно распаковать **внутрь** вашего форка или локальной копии `mmsegmentation`, чтобы получить готовую структуру `practicum_work`, стартовый отчёт `README.md` и вспомогательные скрипты.

---

## Как использовать каркас

1. Сначала скачайте или клонируйте `mmsegmentation`.
2. Затем распакуйте содержимое этого архива **в корень** `mmsegmentation`.
3. Приведите пути к датасету и число классов к своим реальным значениям.
4. Выполните EDA локально через ноутбук и/или скрипты из `practicum_work/src/data`.
5. Подготовьте конфиги стартовых гипотез в `configs/practicum/`.
6. Запускайте эксперименты, логируйте их в ClearML и переносите результаты в этот README.

# Этап 0. Планирование работы

## Цель проекта

Обучить модель семантической сегментации в экосистеме `mmsegmentation` и достичь **mDice > 0.75** на тестовом подмножестве.

## Рабочий план

1. Проверить структуру датасета и целостность пар `image-mask`.
2. Провести EDA:
   - распределение размеров изображений;
   - распределение классов;
   - доля пустых / почти пустых масок;
   - выявление подозрительных или битых разметок.
3. Очистить датасет: удалить проблемные семплы или скорректировать разметку / исключить их из train.
4. Подготовить 1–2 стартовые гипотезы.
5. Провести 2–4 инкрементальных эксперимента по улучшению качества.
6. Выбрать лучший эксперимент и оформить итоговый отчёт.

# Этап 2. Формирование первичных гипотез

стартовые гипотезы такие.
Гипотеза 1 — простой baseline
Идея: взять лёгкую модель и стандартный loss, чтобы получить честную стартовую точку.
Параметры:
модель: SegFormer-B0;
num_classes=3;
loss: CrossEntropy;
аугментации: только horizontal flip;
датасет: cleaned;
метрика: mDice.
Почему это хорошо:
быстро обучается;
удобно проверить весь pipeline;
даёт baseline для сравнения.

Гипотеза 2 — baseline с учётом дисбаланса
Идея: так как фон занимает 90%, обычный CE может переобучаться на фон. Поэтому добавляем Dice Loss или меняем loss на более устойчивый к дисбалансу.
Параметры:
та же модель SegFormer-B0;
те же данные;
те же аугментации;
loss: CE + Dice.
Почему это логично:
Dice напрямую оптимизирует пересечение маски;
лучше подходит, когда объект занимает малую часть изображения.

## Практические замечания

### Можно ли держать всё в одном ноутбуке?

Технически да, но для ревью и воспроизводимости лучше разделить:
- ноутбук: для EDA, быстрых проверок и визуального анализа;
- `.py`-скрипты: для повторяемых действий;
- конфиги: для экспериментов обучения.

### Минимально разумный набор файлов

Если хочется сделать проект без лишней дробности, то можно оставить:
- 1 ноутбук для EDA;
- 4–6 скриптов;
- 2–4 конфига;
- 1 README-отчёт.

Это уже соответствует духу задания и не перегружает проект.
